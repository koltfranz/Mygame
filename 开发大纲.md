## 第零部分：AI 编码绝对红线（前置系统指令）

在生成任何代码之前，你必须在系统提示层锁死以下规则。违背任何一条即视为致命 Error，代码不得提交。

### 红线 1：本体论红线
- **绝对禁止**将“价值”或“使用价值”声明为物品的静态属性。
- ❌ 严禁 `item.value = x` 或 `item.use_value = y`
- ✅ 使用价值通过 `matches_need(commodity, need_type) -> bool` 动态判定，依据 `physical_props` 字典
- ✅ 价值只存在于 `SNLTCalculator` 内部 `current_snlt: Dict[str, float]`，按部门存储，物品本身的 `individual_labor_embodied` 只是个别的私人劳动时间

### 红线 2：行为学红线
- **绝对禁止**在 Agent 的 `step()` 方法中写入基于“贪婪”、“效用最大化”、“避险”等抽象人性假设的条件判断。
- Agent 行为只能通过遍历 `SocialRelationGraph` 的连边决定。
- 例如：工人是否加班不由“我愿意多赚钱”决定，而由 `WageContractEdge` 中资本家设定的工作日长度，以及 `ClassStruggleArbiter` 的博弈结果决定。

### 红线 3：术语红线
- 代码注释、变量名、日志输出必须严格使用附录 A 许可术语表。
- 触及附录 B 禁语表任何一词，必须立即重构。
- 示例：不能用 `money`，必须用 `value_equivalent`；不能用 `profit`，必须用 `surplus_value`。

### 红线 4：机制红线
- 机器绝对不能创造新价值，只能按磨损比例转移旧价值（`c` 转移）。
- 工资是劳动力价值的掩蔽形式，不能等于“劳动报酬”。
- 危机必须由 `p' = s/(c+v)` 下降或 `|I(v+m) - II(c)|` 扩大引起，不能由“消费者信心不足”引发。

### 红线 5：测试红线
- 任何新功能 PR 必须包含至少一个 `tests/test_marxist_laws.py` 中的测试用例，验证未违反上述红线。

### 红线 6：先验性红线
- **绝对禁止**预设复杂劳动的倍加系数（如 `multiplier_watch = 3.0`）。
- **绝对禁止**预设固定技能要求表。技能要求必须由 `Sector.calculate_skill_requirement()` 动态计算，输入为 `tech_knowledge`, `tool_quality`, `avg_worker_skill`。

## 第二部分：核心模型设计（极度详细）

### 2.1 本体论：物品四态机与职能标签

#### 物品状态生命周期
```python
State_Useless -> (劳动) -> State_Product -> (进入市场) -> State_Commodity (Pending)
State_Commodity -> (交换成功) -> 价值实现，物品可能退出或被消费
State_Commodity -> (使用价值丧失/未实现) -> State_Useless (触发价值丧失事件)
State_Product -> (直接消费) -> 物理消耗，退出
```
- `State_Product` 包含 `individual_labor_embodied: float`
- `State_Commodity` 额外包含 `exchange_status: str = "Pending"`，以及 `sector` 属性（通过 `determine_sector` 动态赋予）

#### 物理属性标签 `physical_props: Dict[str, bool]`
用于需求匹配，例如：
`{"edible": True, "wearable": False, "tool": True, "seed": True, "raw_material": False, "luxury": False, "weapon": False}`

#### 职能标签（function_tags）
在物品创建时，根据其 `CommodityType` 赋予初始标签，后续随着使用上下文可能追加（如谷物用作种子时临时标记为 `seed`）。实际部类判定完全由上下文决定：
- `context` 参数值枚举：`"invested_as_seed"`, `"invested_as_raw_material"`, `"consumed_by_worker"`, `"consumed_by_capitalist"`, `"hoarded_as_money"`
- `determine_sector(matter, context) -> str` 返回 `"SECTOR_I"`, `"SECTOR_II"`, `"SECTOR_LUX"`, `"SECTOR_MONEY"`

#### 使用价值丧失
`Matter.use_value_lifespan` 初始值基于物品类型配置，每个时间步（或每次被使用时）减少。归零时触发 `check_use_value_loss()`，若为 `Pending` 商品则发布 `unrealized_commodity_value_loss` 事件。

#### 生产资料磨损与价值转移
当物品标签包含 `tool` 或 `machinery` 时，视为生产资料。额外属性：
- `physical_wear_rate: float`（每单位产出磨损率）
- `idle_wear_rate: float`（每步闲置磨损）
- `remaining_use_value_ratio: float`（初始1.0）
- `moral_depreciation_factor: float`（初始1.0，由部门SNLT变化更新）
- `original_value: float`（购买或生产时记录的价值）
- `transferred_value_accumulated: float`
每生产周期调用 `apply_wear_and_transfer_value()`，返回本期转移价值量，并更新状态。当剩余使用价值或精神磨损因子低于阈值时，标记为 `State_Useless`。

**磨损与各模块的时序联动**：
每个生产周期（资本主义的“生产期”）中，磨损与价值转移发生在劳动之后、SNLT计算之前，具体调用链为：
1. `Human.produce()` 调用 `MeansOfProduction.apply_wear_and_transfer_value(production_quantity)`
2. 该函数更新 `remaining_use_value_ratio` 和 `moral_depreciation_factor`，返回 `value_transferred_this_step`
3. 资本家将 `value_transferred_this_step` 计入本期消耗的 **c**
4. 调用 `SNLTCalculator` 时，消耗的 **c** 已包含磨损转移
5. 利润率计算 `p' = s/(c+v)` 中的 **c** 也包括本期磨损
6. 若机器报废，触发 `means_of_production_scrapped` 事件，资本家需重新购买，影响后续资本有机构成。

**精神磨损触发条件**：当 `SNLTCalculator.update_global_snlt()` 检测到某部门 SNLT 较上期下降 > 5% 时，自动调用 `trigger_moral_depreciation(sector)`，更新该部门所有在用机器的 `moral_depreciation_factor`。

### 2.2 关系边详细定义（relations.py）

所有边继承自 `nx.DiGraph` 的边属性机制。核心边类型及其数据结构：

```python
# 血缘边
KinshipEdge: { kinship_degree: str }

# 交换边
BarterEdge: { trade_count: int, impedance: float }

# 奴隶占有边
EnslavementEdge: { ownership_ratio: float, supervision_cost: float }

# 封建地租边
FeudalRentEdge: { rent_form: str, rent_rate: float, lord_id, serf_id }

# 雇佣边
WageContractEdge: { wage_rate: float, contract_duration: int, capitalist_id, worker_id }

# 培训边
TrainingEdge: { trainer_id, trainee_id, skill_type, remaining_duration, cost_per_day }

# 贡赋边 (酋邦)
TributaryEdge: { tribute_rate: float, chief_id, subject_id }

# 地缘边 (早期国家)
ResidenceEdge: { location_id, resident_ids: list }
```

Agent在 `step()` 中遍历这些边执行相应行为。这强制了行为由结构决定。

### 2.3 Human Agent 详细设计

**核心属性** (在 `agents.py` 中定义)：
- `unique_id`, `model`, `age`, `position`
- `subsistence_satisfaction` (最近一周期的生活资料满足率)
- `labor_power_quality` (健康+技能综合，影响实际有效劳动)
- `skill_level`, `skill_type`, `education_invested`, `training_remaining`
- `labor_power_capacity` (最大日劳动时间)
- `value_equivalent_held` (货币或一般等价物数量)
- `commodity_inventory: list[Matter]`
- `class_position` 属性通过 `infer_class_position()` 动态计算，不在初始化时固定

**主要方法骨架**：
- `produce()`: 根据关系边决定劳动量、使用工具，产出 `Product` 或 `Commodity`
- `exchange()`: 作为卖方或买方参与交换路由
- `consume()`: 从库存中选择物品满足需求，更新 `subsistence_satisfaction`
- `reproduce()`: 根据 `demography` 模块概率生成新 Agent
- `learn()`: 接受培训或干中学，提升技能

每一步的行为都是被动的：例如 `produce()` 会读取自己所有的 `WageContractEdge` 获得要求的工作时长，读取雇主分配的 `MeansOfProduction`，然后执行，而非自主决定。

**需求与消费逻辑（核心实现）**：

Agent 的消费过程严格遵循“再生产强制”，算法如下：

```python
def get_required_consumption_basket(agent):
    """
    返回：{need_category: {tag: required_quantity}}
    """
    class_name = agent.infer_class_position()
    stage = agent.model.social_stage
    basket = config.class_demands[stage][class_name].copy()

    # 历史社会必要需求扩展
    basket.update(config.historical_social_necessities.get(stage, {}))

    # 复杂劳动力再生产：教育摊销转换为额外消费品需求
    if agent.skill_level > 1.0 and agent.education_invested > 0:
        amort_daily = agent.education_invested / agent.expected_remaining_workdays
        extra = amort_daily / avg_price_of_consumer_goods
        basket['food']['qty'] += extra * 0.5   # 额外食物
        basket['cultural']['qty'] += extra * 0.5  # 书报等
    return basket

def can_satisfy_need(commodity, need_category):
    required_tags = config.need_tags[need_category]
    return any(commodity.physical_props.get(tag, False) for tag in required_tags)

def consume(agent):
    basket = get_required_consumption_basket(agent)
    inv = agent.commodity_inventory
    satisfaction = {}
    for need_cat, req in basket.items():
        needed = req['qty']
        consumed = 0.0
        suitables = [i for i in inv if can_satisfy_need(i, need_cat)]
        suitables.sort(key=lambda x: x.use_value_lifespan)  # 临期先消费
        for item in suitables:
            if consumed >= needed: break
            take = min(item.quantity, needed - consumed)
            item.quantity -= take
            consumed += take
            if item.quantity <= 0:
                inv.remove(item)
        satisfaction[need_cat] = consumed / needed if needed > 0 else 1.0

    agent.subsistence_satisfaction = min(satisfaction.values())
    if agent.subsistence_satisfaction < 0.5:
        agent.labor_power_quality *= 0.9
        agent.revolt_potential += 0.1
    # 文化需求影响阶级意识
    if satisfaction.get('cultural', 0) > 0.8:
        agent.class_consciousness += 0.005
```

**关键约束**：禁止效用函数，标签匹配纯用 `physical_props`。

class Sector:
    def __init__(self, name, commodity_type, initial_config):
        self.name = name
        self.commodity_types = commodity_type
        self.tech_knowledge = initial_config['tech_knowledge']
        self.tool_quality = initial_config['tool_quality']

        self.accumulated_output = 0.0

    def step(self, production_volume, r_and_d_investment, machines_in_use):
        self.tech_knowledge += production_volume * config.LEARNING_RATE + r_and_d_investment * config.R_AND_D_EFFICIENCY
        if machines_in_use:
            self.tool_quality = np.mean([m.snlt_value for m in machines_in_use])
        # 平滑更新平均技能...

    def calc_skill_requirement(self):
        raw = self.tech_knowledge / (1 + self.tool_quality)
        return max(1.0, 0.7 * raw + 0.3 * self.avg_worker_skill)

## 第三部分：经济引擎核心算法（详细）

### 3.1 SNLT 事后计算器（labor_value.py）
每步在交换前调用，遍历 `Pending` 商品，按 `sector` 分组平均，存入 `self.current_snlt`，不写回物品。监听未实现价值丧失事件移除商品。

### 3.2 阻抗路由与价值形式涌现（value_form_router.py）
- 物物交换：需求双重巧合匹配，成功降低阻抗。
- 阻抗定义：`base_distance_impedance + information_impedance`。
- 货币涌现：某节点入度中心性 > 80% 且持续，自动成为一般等价物，不预设。

### 3.3 再生产与利润率平均化（reproduction.py）
- 每周期计算部类价值，输出偏离度。
- 利润率差异触发资本流动，形成平均利润率，计算生产价格用于跨部门交换。

### 3.4 阶级斗争模块（class_struggle.py）
- 计算潜在 `v` 下降，`Resist_Force` 依赖失业率、工会密度、阶级意识。
- 高抵抗触发罢工。

### 3.5 复杂劳动涌现的完整数值推演

**假设**：谷物 SNLT=1.0h/单位，手表 10h/只，芯片 2h/枚，各自 c 不同。
利润率平均化后生产价格：
- 谷物: 0.72, 手表: 9.6, 芯片: 25.2
事后倍加系数：
- 手表 = (9.6/0.72) / (10/1) ≈ 1.33
- 芯片 = (25.2/0.72) / (2/1) = 17.5

此系数在模拟中成功涌现。

### 物品状态跃迁与使用价值丧失
```python
class Matter:
    def __init__(self):
        self.state = State_Useless
        self.physical_props = {}
        self.individual_labor_embodied = 0.0
        self.use_value_lifespan = 100
        # 生产资料附加
        if self.physical_props.get("means_of_production"):
            self.physical_wear_rate = 0.01
            self.idle_wear_rate = 0.001
            self.remaining_use_value_ratio = 1.0
            self.moral_depreciation_factor = 1.0
            self.original_value = 0.0
            self.transferred_value_accumulated = 0.0

    def check_use_value_loss(self):
        self.use_value_lifespan -= 1
        if self.use_value_lifespan <= 0:
            if self.state == State_Commodity and self.exchange_status == "Pending":
                EventBus.emit("unrealized_commodity_value_loss", self)
            self.state = State_Useless
            self.individual_labor_embodied = 0.0
```

### 生产资料磨损与价值转移
```python
def apply_wear_and_transfer_value(machine, production_quantity):
    total_wear = machine.physical_wear_rate * production_quantity + machine.idle_wear_rate
    current_snlt = SNLTCalculator.get_snlt(machine.sector)
    moral_factor = current_snlt / machine.original_snlt_at_production
    machine.moral_depreciation_factor = moral_factor
    machine.remaining_use_value_ratio -= total_wear
    effective_wear = 1 - (machine.remaining_use_value_ratio * moral_factor)
    transferred = machine.original_value * effective_wear - machine.transferred_value_accumulated
    machine.transferred_value_accumulated += transferred
    if machine.remaining_use_value_ratio <= 0 or moral_factor <= 0.01:
        machine.state = State_Useless
        EventBus.emit("means_of_production_scrapped", machine)
    return transferred
```
