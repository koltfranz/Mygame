## 第零部分：AI 编码绝对红线（前置系统指令）

在阅读后续任何技术需求之前，你必须在系统提示层锁死以下规则，违背任何一条即视为致命 Error：

### 红线 1：本体论红线
**绝对禁止**将“价值”或“使用价值”声明为物品的静态属性。
- ❌ 严禁 `item.value = x` 或 `item.use_value = y`
- ✅ 使用价值是动态判定事件（`matches_need()` 函数）
- ✅ 价值是事后结算的幽灵比例（`SNLTCalculator.current_snlt`）

### 红线 2：行为学红线
**绝对禁止**在 Agent 的 `step()` 方法中写入任何基于“贪婪、效用最大化、避险”等抽象人性的 if-else 逻辑。
- Agent 行为必须且只能从 `SocialRelationGraph` 的连边推导
- 禁止出现 `if agent.wants_more_money:` 之类的代码

### 红线 3：术语红线
代码注释、变量命名、日志输出必须严格使用附录 A 的许可术语表。
- 触及附录 B 禁语表任何一词，必须立即重构
- 变量名如 `money`、`profit`、`price`、`decay`、`rot` 一律禁止

### 红线 4：机制红线
- 机器绝对不能创造新价值（只能转移 $c$）
- 工资绝对不能等于劳动的报酬（只能是劳动力价值的掩蔽形式）
- 危机绝对不能由主观需求不足引起（只能由 $p' = s/(c+v)$ 下降或再生产比例失衡引起）

### 红线 5：测试红线
任何新功能或修改的 Pull Request 必须附带至少一个位于 `tests/test_marxist_laws.py` 中的测试用例，验证本次变更未违反上述四条红线。

### 红线 6：先验性红线
- **绝对禁止**预设复杂劳动的倍加系数（如 `if sector=="watch": multiplier=3.0`）
- **绝对禁止**预设固定技能要求表
- 技能要求必须由技术知识、工具质量和劳动者技能分布动态计算

## 第一部分：项目定义、技术选型与理论约束

### 1.1 项目信息

- **名称**：CapitalSimulator  
- **类型**：历史唯物主义经济模拟 / Agent-Based 涌现式沙盒  
- **核心体验**：观察从原始群到社会主义的全历史阶段中，无数个体在特定生产关系下的结构性强制决策，如何涌现出阶级、价值形式、货币、国家、法律、意识形态、危机与革命  
- **理论依据**：
  - 马克思《资本论》全三卷（价值形式表、再生产图式、利润率趋向下降规律、生产价格理论、原始积累）

  - 广松涉“物象化论”
  - 塞维斯-弗里德“游群-部落-酋邦-国家”人类学框架  
- **模拟范式**：Agent-Based Modeling (ABM)，基于 Mesa 框架与 NetworkX 图论引擎

## 第二部分：Mesa 模型与理论核心设计

### 2.1 底层本体论：物品四态机与生产资料磨损

任何物质团块在内存中必须处于以下四种状态之一，由 `ontology.py` 严格控制：

1. **`State_Useless` (无用物)**：物理上无法满足当前人类需求。
2. **`State_Pure_Use_Value` (纯使用价值)**：自然界直接提供、无劳动介入即满足需求之物。
3. **`State_Product` (产品)**：凝结了人类具体劳动的有用物。具有 `physical_props` 和 `individual_labor_embodied`。
4. **`State_Commodity` (商品)**：产品跨越社会边界的跃迁态。包含 `exchange_status="Pending"`。

**物品职能标签**：

| 标签 | 含义 |
|:---|:---|
| `edible` | 可食用 |
| `seed` | 可作为种子 |
| `tool` | 可作为工具 |
| `raw_material` | 可作为原料 |
| `luxury` | 奢侈品属性 |
| `money_commodity` | 货币商品属性 |
| `weapon` | 武器 |
| `location_change` | 运输服务 |

**部类动态判定**：`determine_sector(matter, context)` 根据上下文返回 `SECTOR_I`（生产资料）、`SECTOR_II`（消费资料）、`SECTOR_TRANSPORT`（运输）或 `SECTOR_MONEY`（货币商品）。

**使用价值丧失**：
- `use_value_loss_check()`：物品因时间或使用失去有用性，状态降为 `State_Useless`。
- 若商品处于 `Pending` 状态且发生使用价值丧失，触发 `unrealized_commodity_value_loss` 事件，SNLT计算器将其移出待结算池。

**生产资料磨损与价值转移**：
- **有形磨损**：由使用和自然力引起的使用价值丧失，价值按磨损比例转移。
- **无形磨损（精神磨损）**：因该部门 SNLT 下降导致的原有资本贬值。
- **价值转移量** = 原始价值 × (1 - 剩余使用价值比例 × 精神磨损系数) - 累计已转移价值。

**初始化**：各阶段阶级比例配置于 `config/initial_conditions/`。

**动态生成与消亡**：
- 出生/死亡由 `population.demography` 模块计算。
- 移民：国内基于城乡工资差，国际流向殖民地。

**Mesa 调度器策略**：
- 非资本主义阶段：`RandomActivation`
- 资本主义阶段：`CapitalistStagedScheduler` 强制五步时序（生产→SNLT→交换→阶级博弈→淘汰）

### 2.4 部门与技能要求动态演化

部门技能要求完全内生演化：

```python
def calculate_skill_requirement(sector):
    raw_req = sector.tech_knowledge / (1 + sector.tool_quality)
    social_anchor = 0.7 * raw_req + 0.3 * sector.avg_worker_skill
    return max(1.0, social_anchor)
```

### 2.5 空间设计

- **基础空间**：`mesa.space.ContinuousSpace`
- **地块逻辑**：`Landscape` 类维护物理层和社会层
- **交互距离**：受运输技术影响，市场中心作为涌现的空间节点降低阻抗

### 2.6 时间调度与严格时序切片

- **基本步长**：1 季度（早期）→ 1 年（后期）
- **资本主义五步时序（不可颠倒）**：
  1. 生产期：工人消耗劳动力，产出 `Commodity` (Pending)
  2. SNLT 事后裁决
  3. 阻抗路由与强制交割
  4. 阶级博弈期
  5. 淘汰期

### 2.7 数据收集指标

必须收集以下指标：
- 总人口、各边类型数量、平均生活资料满足率、基尼系数
- 网络中心性指标（一般等价物入度垄断率）
- 利润率 $p'$、部类资本有机构成 $C/V$、产业后备军率、部类偏离度 $|I(v+m) - II(c)|$
- 事后倍加系数、贸易顺差/逆差、殖民地净收益、军费占剩余价值比
- 人口增长率、总和生育率、移民净流出量
- 法律形式平等指数、产权保护强度、政体类型、合法性分数、文化霸权强度
- 当前社会阶段、世袭权力指数、地域划分比例、物象化指数

### 2.8 社会演进阶段与时间轴

本模拟器采用**塞维斯-弗里德**框架扩展版：

| 阶段代码 | 人口规模 | 时间尺度 | 核心关系边 | 权力结构 |
|:---|:---|:---|:---|:---|

| `TRIBE` | 100-500 | 10→1年/步 | `ClanEdge` | 议事会民主 |

| `CHIEFDOM` | 2000-10000 | 1年/步 | `TributaryEdge` | 世袭酋长 |
| `EARLY_STATE` | 10000+ | 1年/步 | `ResidenceEdge` | 官僚+常备军 |
| `SLAVERY_STATE` | 可变 | 1年/步 | `EnslavementEdge` | 奴隶主专政 |
| `FEUDAL_STATE` | 可变 | 1年/步 | `FeudalRentEdge` | 地主阶级统治 |

**转型条件示例**：
- 原始群→游群：火的使用 + 语言能力
- 游群→部落：定居率 > 30% + 植物驯化知识
- 部落→部落联盟：剩余产品率 > 10% + 战争频率 > 阈值
- 部落联盟→酋邦：军事首领世袭化 + 基尼系数 > 0.4
- 酋邦→早期国家：地域划分 > 血缘划分 + 公共权力制度化

```python
class Matter:
    def __init__(self):
        self.state = State_Useless
        self.physical_props = {}
        self.individual_labor_embodied = 0.0
        self.exchange_status = None
        self.use_value_lifespan = 100
        # 生产资料特有属性
        if self.physical_props.get("means_of_production"):
            self.physical_wear_rate = 0.01
            self.idle_wear_rate = 0.001
            self.remaining_use_value_ratio = 1.0
            self.moral_depreciation_factor = 1.0
            self.original_value = 0.0
            self.transferred_value_accumulated = 0.0

    def check_use_value_loss(self):
        self.use_value_lifespan -= 1
        if self.use_value_lifespan <= 0:
            if self.state == State_Commodity and self.exchange_status == "Pending":
                EventBus.emit("unrealized_commodity_value_loss", self)
            self.state = State_Useless
            self.individual_labor_embodied = 0.0
```

### 5.2 生产资料磨损与价值转移

```python
def apply_wear_and_transfer_value(machine, production_quantity):
    physical_wear = machine.physical_wear_rate * production_quantity
    idle_wear = machine.idle_wear_rate
    total_wear = physical_wear + idle_wear

    current_snlt = SNLTCalculator.get_snlt(machine.sector)
    original_snlt = machine.original_snlt_at_production
    moral_factor = current_snlt / original_snlt
    machine.moral_depreciation_factor = moral_factor

    machine.remaining_use_value_ratio -= total_wear

    effective_wear = 1 - (machine.remaining_use_value_ratio * moral_factor)
    value_transferred = machine.original_value * effective_wear - machine.transferred_value_accumulated
    machine.transferred_value_accumulated += value_transferred

    if machine.remaining_use_value_ratio <= 0 or moral_factor <= 0.01:
        machine.state = State_Useless
        EventBus.emit("means_of_production_scrapped", machine)

    return value_transferred
```
